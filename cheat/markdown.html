标题#

标题分为多行标题（两种，文字下方加=或-）和单行标题（六种，行首加多个#）。它们只是书写方式不同，实际显示效果是一样的。

多行标题一
=====

多行标题二
-----

# 单行标题一
## 单行标题二
### 单行标题三
#### 单行标题四
##### 单行标题五
###### 单行标题六

段落#

连续的几行显示在一个段落内，使用空行分段。

第一个段落，
还是第一个段落

第二个段落

第三个段落

引用#

在段落前面加>，或者在每一行前面加>，表示引用内容。引用可以嵌套。

提示：在 Markdown 中，很多以起始符开头的语法，在起始符前面都可以留 1-3 个空格，但 4 个空格将被视作代码块。

> 引用的段落
>
> > 更深层次的引用

列表#

列表分为无序列表和有序列表两种。无序列表以+、-、*开头，有序列表以数字开头，后接一个.。起始符后面需要接至少一个空格或制表符。列表可以嵌套。

提示：有序列表的序号是根据第一个序号的数字往后顺序排列的，后面的数字完全相同也无所谓。

无序列表：

* 列表项目
* 列表项目
* 列表项目

有序列表

2. 列表项目
2. 列表项目
   * 嵌套列表
   * 嵌套列表
2. 列表项目

任务列表（GFM）#

GitHub Flavored Markdown 支持任务列表。任务列表首先是一个列表，它的项目以[ ]（注意，中间有空格）表示未完成，以[x]表示已完成。

注意：传统意义上，任务列表使用<input type="checkbox">标记元素。但因为该解析方式与 Spectre.css 的默认样式不兼容也不美观，所以记事本对此重新进行了定义，直接使用 Spectre.css 的label和icon样式。

1. [x] 任务一
1. [x] 任务二
1. [ ] 任务三

代码块#

代码块使用 4 个空格符或 1 个制表符开头即可。当然，更简单的方法是使用围栏式代码块，以```开始首行（可以在后面加语言代码），并在最后以```结束。

注意：由于 Markdown 标准与 Spectre.css 的默认样式并不兼容，导致无法完整应用 Spectre.css 的样式，所以记事本依照 Spectre.css 的样式重新进行了定义。

```ruby
def foo(x)
    return 3
end
```

表格（GFM）#

GitHub Flavored Markdown 支持表格。表格的列元素以|分割，并且第二行需要有以-表示的分隔符。可选的，用:放在左边，表示左对齐，或放右边，表示右对齐，或两边都放，表示居中对齐。

注意：由于 Markdown 标准与 Spectre.css 的默认样式并不兼容，导致无法完整应用 Spectre.css 的样式，所以记事本依照 Spectre.css 的样式重新进行了定义。

表头一|表头二|表头三
:---|:--:|---:
左对齐|居中对齐|右对齐
单元格一|单元格二|单元格三

分隔线#

使用 3 个以上的-、_、*建立分割线，中间可以插入空格，但是不要有其它的内容。

注意：传统意义上，分隔线使用<hr>标记。但因为不太美观，所以记事本对此重新进行了定义，直接使用 Spectre.css 的divider样式。

强调#

有两种强调方式，使用*或_包裹文字表示<em>强调，使用**或__包裹文字表示<strong>强调。

注意：传统意义上，<em>通常显示为斜体，<strong>通常显示为加粗。但因为这两种显示方式都不太美观，所以记事本对此重新进行了定义，将<em>改为<mark>，<strong>改为<ins>。

此外，GFM 还支持删除线，用~~包裹文字即可。

*em 强调*

_em 强调_

**strong 强调**

__strong 强调__

~~删除线~~

链接#

链接一般分为行内式和参考式两种。行内式链接的格式为[文字](链接)，一般是一次性使用。参考式链接的格式为[文字][编号]，然后在文章内（文章底部或段落结尾处都可以）用单独的段落定义这个[编号]代表的链接地址[编号]: 链接。参考式便于在文章内多次使用。

提示：链接支持本地相对路径。

此外，GFM 还支持以 www 开头的网址自动转换为链接，但复杂的网址是否能被正确转换还请自行确认。

[百度](https://www.baidu.com/)

[文档模板](../docs/document.html)

[淘宝][taobao]
也是[淘宝网][taobao]

[taobao]: https://www.taobao.com/

www.qq.com

图片#

图片与链接非常像，只是前面要加一个!，而且也分为行内式和参考式两种。行内式图片的格式为![文字](图片地址)，一般是一次性使用。参考式图片的格式为![文字][图片编号]，然后在文章内（文章底部或段落结尾处都可以）用单独的段落定义这个[图片编号]代表的图片地址[编号]: 图片地址。参考式便于在文章内多次使用。

注意：图片都是根据实际大小显示，无法重新定义大小。

![图片](http://placehold.it/150&text=image)

![图标][icon]
也是![图标][icon]

[icon]: http://placehold.it/60&text=icon

行内代码#

行内代码使用一个或多个`包裹文字即可。

`foo`

`` foo ` bar  ``

反斜线#

反斜线用于将后面的符号原文呈现。例如*原本是表示强调，但使用\*就可以呈现原始的*了。

提示：HTML 字符实体（如>、&等）在 Markdown 中会自动转化，所以不需要使用反斜线。

\*原文呈现\*

记事本定制#

如前所述，由于 Markdown 的标准解析方式与 Spectre.css 的默认样式并不兼容，导致无法完整应用 Spectre.css 的样式。而且即使按照标准的 Markdown 作出解析，转换出来的文字也不一定不好看（如中文斜体），所以记事本对 Marked.js 的部分默认解析方式重新进行了定义，包括：任务列表、代码块、表格、分隔线、强调。

此外，由于在记事本中使用 Markdown 时，需要用 HTML 注释<!-- ... -->包裹 Markdown 代码块（参见文档模板）所以为了保证在 Markdown 中仍能表达 HTML 注释，记事本对 Markdown 中的 HTML 注释语法进行了转义，即用<!-- ... --!>代替<!-- ... -->（注意结尾处多出来的!）。

```HTML
<!-- ... --!>
```
``` -->
