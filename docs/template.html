静态页面#

静态页面static.html内不包含任何动态内容，其编辑文档的格式是HTML，这也是记事本的默认编写方式。记事本使用Spectre.css作为默认的文档样式，您可以访问Spectre.css 官网了解这些预设的 HTML 标记和样式。

由于纯静态页面很难处理重复性内容，而且很难扩展，故使用时会非常不方便。该页面一般仅用作存档，日常工作中通常使用文档模板作为编辑内容时的起始模板。

文档模板#

文档模板document.html实现了编辑文档的基本功能，包括对重复性内容（如左侧菜单栏）的处理（配置文件jac/config.js），并基于Marked.js实现了对Markdown类型文本的处理，您可以访问Marked.js 官网查阅帮助文档。另外，Marked.js 默认支持 GitHub Flavored Markdown 格式，其语法可以在GFM 官方说明找到。速查手册中也有关于Markdown 语法的详细说明。

在段落中使用 Markdown 时，应使用class="markdown"标记段落（默认值，可以在配置文件中修改）。并且为了避免 Markdown 对 HTML 代码的干扰，应使用 HTML 注释<!-- ... -->包裹 Markdown 代码块，其基本结构为：

<section class="markdown">
<!-- NOTE: 一句话说明，没有也无所谓

这里是 Markdown 格式的正文。

-->
</section>

鉴于文档模板的便捷性，您在编写文字时，可以用文档模板作为起始模板，复制一个新文件，重命名，并在它的基础上进行修改。

注意：文档模板中的路径都是从docs目录开始算起。如果新建的文档不在docs目录中，则使用文档模板作为起始模板时，记得同时修改所有与路径相关的内容，如href、src属性等。

提示：您也可以在其它目录中各自做一份文档模板，作为该目录中文件的默认模板。

多步页面模板#

多步页面模板step.html用于在单一页面内将一项工作分为少数几个依次递进的工作步骤，并对这些工作步骤分别展开说明。

当工作步骤太多时，使用单一的多步页面也许并不合适，这时您可以改为选用时间轴模板，或者干脆将它们分为几项工作，并分别在多个页面内予以说明。

日历模板#

日历模板calendar.html实现了简单的日历牌和日历标注功能。日历牌以表格样式（实际不是表格）显示一个月内的公历信息，日历标注功能允许您在日历牌上直接标注一些记录事项。

您虽然可以直接在日历上标注一些信息，但受限于页面宽度，可以记录的信息实际上并不多。这时您可以另起一段文字补充说明，也可以选择与时间轴模板建立链接，这样就可以从日历牌信息跳转到时间轴内的某一具体记录。参考日历模板的演示内容。

注意：日历模板尚未实现动态生成功能。

时间轴模板#

时间轴模板timeline.html依据时间轴分步骤显示工作进程。由于时间轴采用的是纵向排列，它可以记录的项目非常多。

虽然名为时间轴，但实际上只要是分步骤依次递进的过程，都可以采用时间轴作为模板。如之前提到的多步页面模板和日历模板等，都可以通过某种方式转换为时间轴模板，以便容纳大量的信息。

提示：时间轴本身容纳的信息已经很多了，所以它通常不需要更多的补充信息。这时，跳转菜单可以直接定位到时间轴的定位点上，以便用户快速跳转和浏览。

筛选页面模板#

筛选页面模板filter.html实现多个乱序项目的分组筛选功能。筛选页面使用tag-1到tag-8可标记八种不同的标签，使用tag-0显示全部项目（取消筛选）。具体可直接参见筛选页面模板的演示内容。
